The Transformer Wizard can be used to transform an existing model to a target XML Schema.

# Transform an existing Model #
  1. Prepare a UPCC3 model in Enterprise Architect
  1. Select _Add-Ins / VIENNAAddIn / Wizards / Transformer Wizard_ from the main menu:
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild01.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild01.gif)
  1. The Transformer Wizard guides you through the whole transformation process step by step. You can mover your mouse over every icon representing a step in the transformation process to view additional information.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild02.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild02.gif)
  1. The first step is selecting the Source Model, if you hover the icon you'll receive a hint saying you have to select a BIE and a DOC library.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild03.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild03.gif)
  1. After clicking on the icon, a selection screen pops up and you can select the source libraries.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild04.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild04.gif)
  1. In this example we select the BIE library of ebInterface.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild05.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild05.gif)
  1. Next we have to define the source DOC library, again we choose the one containing ebInterface.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild06.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild06.gif)
  1. After completing this step, the source model step icon will be coloured, indicating that this step is completed. When you hover the icon you can see the current selection.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild07.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild07.gif)
  1. Now it is time to define a target XML Schema which can be either selected by clicking on the icon or by drag & drop the target file.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild08.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild08.gif)
  1. This is the selection screen following after a click on the icon.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild09.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild09.gif)
  1. You can choose the target file using a standard Windows file dialog.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild10.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild10.gif)
  1. The alternative method using drag & drop is visualized below:
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild11.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild11.gif)
  1. Now the target XSD step should be also coloured, indicating that this step is completed and showing the selection after hovering the icon.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild12.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild12.gif)
  1. Next you have to choose the target model, which again consists of a BIE and a DOC library.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild13.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild13.gif)
  1. Like in the Source Model step before, a screen pops up this time asking for the target libraries.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild14.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild14.gif)
  1. This time we choose the UBL BIE library for our example transformation.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild15.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild15.gif)
  1. The DOC library has to be defined to complete this step.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild16.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild16.gif)
  1. We now choose the UBL Doc library to complete our example transformation.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild17.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild17.gif)
  1. If the Target Model icon is coloured, this step is also completed and showing the current selection as tooltip.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild18.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild18.gif)
  1. The last step is to define a output directory. This can again be done by clicking on the icon or using drag & drop.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild19.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild19.gif)
  1. When you click on the icon, the directory selection screen pops up.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild20.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild20.gif)
  1. You can select the target folder using a standard windows dialog.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild21.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild21.gif)
  1. As an alternative you can drag & drop any folder onto the icon.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild22.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild22.gif)
  1. A coloured icon again symbolises the completation of this step.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild23.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild23.gif)
  1. Now we are ready to start the transformation. You just have to click the Generate button.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild24.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild24.gif)
  1. During the transformation process a spinning wheel will indicate progress.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild25.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild25.gif)
  1. If the process was successfull a notification is displayed, offering to open the resulting model.
> > ![http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild26.gif](http://vienna-add-in.googlecode.com/svn/wiki/images_transformerwizard/Bild26.gif)
